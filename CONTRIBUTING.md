# Contributing

1. Start from the frozen baseline.
2. Use one branch for one objective.
3. Inspect `git status` before work.
4. Do not change unrelated architecture.
5. Add or update an ADR for structural decisions.
6. Run the test plan.
7. Update the changelog, audit, optimization log and handoff.
8. Include rollback instructions.
9. Do not commit secrets.
10. Do not merge changes without test evidence.

Suggested branch names:

```text
feature/production-auth
feature/translation-endpoint
feature/cloudinary-upload
feature/bisheng-assistant
fix/building-wall-mobile
```
