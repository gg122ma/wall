# Echo Wall 后续功能与开发规划

## 一、正式方向

继续采用现有架构：

- HTML
- CSS
- Vanilla JavaScript
- Hash Router
- Leaflet
- LocalStorage 原型数据

不重写成 React、Next.js、Vue 或 TypeScript。KMK Digital Twin 只作为建筑资料与区域划分参考，Echo Wall 运行时不依赖孪生项目。

地图区域整改暂缓，当前优先完成账号、建筑留言墙、语言、主题、便贴和外部服务对接基础。

## 二、用户主要流程

```text
建筑列表
→ 建筑简介
→ 建筑俯瞰图与热门留言
→ 进入该建筑专属留言墙
→ 登录后留言
→ 选择匿名或显示名称
→ 选择便贴位置、形状与照片裁切方式
```

建筑留言墙无需选择 Batch 或 Major。

## 三、本版本已经完成

### 1. 建筑简介与建筑专属留言墙

- 建立 30 个 KMK 建筑资料。
- 新增建筑列表页面。
- 新增建筑简介页面。
- 新增每栋建筑独立留言墙。
- 建筑简介显示说明、区域、楼层、开放时间和标签。
- 热度最高的可见留言显示在建筑俯瞰轮廓上。

### 2. 注册与登录

- 增加本地原型注册和登录。
- 未登录用户不能在社区墙、建筑墙或地图发布留言。
- 已登录用户可以选择匿名或显示账号名称。
- 当前账号系统只适合本地原型，正式上线必须替换为 Supabase Auth 或其他服务器端身份系统。

### 3. 便贴系统

- 用户可以在发布前选择便贴放置位置。
- 新增圆角、对话框、拍立得、票券和六边形等形状。
- 保留原有正方形、长方形、圆形、信封和撕边。
- 桌面端按照用户选择的位置显示。
- 手机端继续使用安全的单栏排列，避免便贴互相覆盖。

### 4. 照片裁切

- 保留上传前压缩。
- 用户可以选择 100%–180% 裁切缩放。
- 用户可以选择“裁切填满”或“显示完整照片”。
- 已准备 Cloudinary signed upload 适配器。

### 5. 语言系统

语言文件独立放在：

```text
i18n/
  index.js
  locales/en.js
  locales/ms.js
  locales/zh.js
```

支持：

- English
- Bahasa Melayu
- 华文

网站核心页面会根据用户选择切换语言，选择会保存到浏览器。

### 6. 留言翻译

- 留言弹窗新增“翻译 / 查看原文”切换。
- 翻译结果会缓存。
- 目标语言跟随网站语言。
- 真实翻译需要连接后端翻译接口。

### 7. 明亮与黑暗模式

支持：

- 明亮模式
- 黑暗模式
- 跟随系统

主题选择会保存到浏览器。

### 8. BISHENG 对接准备

已经建立独立桥接文件：

```text
services/bisheng-adapter.js
```

后续组员只需要提供：

- BISHENG 部署接口
- App ID
- 凭证处理方式
- 对话返回格式

建议通过后端代理连接，避免把私密凭证放进浏览器。

### 9. Cloudinary 对接准备

已经建立：

```text
services/cloudinary-adapter.js
```

流程为：

```text
浏览器压缩图片
→ 后端签名接口
→ 浏览器直传 Cloudinary
→ 保存 secure URL 与 public ID
```

Cloudinary API Secret 不会放进前端。

## 五、每个阶段的强制要求

每次修改必须包含：

- 修改目标
- 修改文件
- 修改前后行为
- 测试项目和结果
- 已知限制
- 数据迁移说明
- 回滚方法
- 更新日志
- 交接说明

其他未获确认的架构不得同时修改。
